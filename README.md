# 2048 小游戏 - 功能说明文档

## 📖 项目简介

这是一个基于原生 HTML、CSS 和 JavaScript 开发的 2048 小游戏，提供了完整的游戏功能和流畅的用户体验。

## 🎮 核心功能

### 1. 实时计分系统
- **分数显示**：在界面顶部实时显示当前分数和历史最高分
- **得分飘字**：每次合并方块时，会在棋盘中央显示得分飘字动画
- **动态更新**：分数实时刷新，无需手动刷新

### 2. 最高分本地持久化
- **自动保存**：每次游戏结束后自动保存最高分到 localStorage
- **数据持久**：页面刷新后最高分数据不会丢失
- **自动读取**：游戏启动时自动读取历史最高分

### 3. 难度选择功能
游戏提供三个难度级别：

| 难度 | 网格大小 | 特点 | 适合人群 |
|------|---------|------|---------|
| 简单 | 5×5 | 格子多，空间大 | 新手玩家 |
| 普通 | 4×4 | 经典模式 | 大多数玩家 |
| 困难 | 3×3 | 格子少，空间紧凑 | 挑战高手 |

- 切换难度后会自动重置游戏
- 难度选择位于标题下方

### 4. 完整游戏流程控制

#### 控制按钮
- **开始游戏**：初始化新游戏，生成两个初始方块
- **暂停游戏/继续游戏**：一键暂停/继续，暂停时显示遮罩提示
- **重新开始**：重置当前游戏（会弹出确认提示）
- **排行榜**：打开本地排行榜弹窗

#### 游戏状态管理
- 游戏初始状态为未开始，需点击"开始游戏"按钮
- 游戏进行中可以随时暂停
- 游戏结束后自动记录到排行榜

### 5. 本地排行榜
- **自动记录**：每局游戏结束后自动保存分数
- **排序展示**：按分数降序显示前10名记录
- **详细信息**：显示排名、分数、难度等级、游戏日期
- **局数统计**：显示总游戏局数
- **清空功能**：可一键清空所有历史记录
- **数据持久**：保存在 localStorage，刷新不丢失

### 6. 移动端适配优化
- **触屏滑动**：支持手指滑动操作，流畅度优化
- **响应式布局**：适配手机、平板、桌面端
- **触摸优化**：按钮最小高度 44px，符合移动端点击标准
- **视口设置**：禁用缩放，优化移动端体验

## 🎯 游戏规则

1. 使用方向键或触摸滑动控制所有方块移动
2. 相同数字的方块碰撞时会合并成它们的和
3. 每次移动后随机生成一个新的 2 或 4
4. 当达到 2048 时游戏胜利
5. 当无法移动时游戏结束

## 💻 操作方式

### 桌面端
- **↑** 向上移动
- **↓** 向下移动
- **←** 向左移动
- **→** 向右移动

### 移动端
- 在游戏区域向上/下/左/右滑动

### 控制按钮
- 点击相应按钮进行游戏控制

## 🛠️ 技术实现

### 代码结构
项目采用模块化架构，分为以下几个模块：

```
game.js
├── StorageModule      # 存储模块（localStorage操作）
├── ScoreModule        # 计分模块（分数管理）
├── AnimationModule    # 动画模块（得分飘字等）
├── LeaderboardModule  # 排行榜模块（UI和数据）
├── GameControlModule  # 游戏控制模块（流程控制）
└── GameCore          # 游戏核心模块（主逻辑）
```

### 技术特点
- **原生开发**：纯 HTML + CSS + JavaScript，无第三方框架
- **本地存储**：使用 localStorage 实现数据持久化
- **响应式设计**：CSS 媒体查询适配不同屏幕
- **动画效果**：CSS transitions 和 keyframes 动画
- **触摸支持**：touch 事件实现移动端滑动

## 📦 部署说明

项目可直接部署到 GitHub Pages：

1. 将 `index.html`、`style.css`、`game.js` 上传到 GitHub 仓库
2. 进入仓库 Settings → Pages
3. 选择 branch 为 `main` 或 `master`
4. 访问生成的 URL 即可玩游戏
